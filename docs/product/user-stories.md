# User Stories

## Patient Stories

### Access And Trust

- As a user, I want to create an account easily so I can book a session without extra friction.
- As a user, I want to sign in securely and stay signed in so I can return to my bookings quickly.
- As a user, I want the app to feel discreet and professional so I feel comfortable using it for mental health care.

### Discovery

- As a user, I want to browse therapists by relevant information so I can find someone who matches my needs.
- As a user, I want to see therapist specialization, language, and session type so I can choose with confidence.
- As a user, I want a therapist detail page with enough information to decide whether to book.

### Booking

- As a user, I want to see available time slots so I can book a session that fits my schedule.
- As a user, I want to review the booking details before I pay so I can confirm I selected the right therapist and slot.
- As a user, I want clear confirmation after booking so I do not wonder whether the appointment was created successfully.

### Payment

- As a user, I want to see the session price before payment so I know the cost upfront.
- As a user, I want to pay through a payment flow that feels familiar in Kuwait so I trust the transaction.
- As a user, I want the app to tell me clearly if payment failed so I know whether I need to retry.

### Session Attendance

- As a user, I want to join my session from inside the app so I do not need to switch tools at a stressful moment.
- As a user, I want to know when the join button becomes available so I do not try too early or miss the session.
- As a user, I want audio-only support when needed so I can still attend even in lower-comfort or lower-bandwidth situations.

### History And Account

- As a user, I want to see my upcoming bookings so I can stay organized.
- As a user, I want to review my past bookings and statuses so I understand what happened and what I paid for.
- As a user, I want to edit my basic profile details so my booking information stays accurate.

## Therapist Stories

- As a therapist, I want my public profile to show accurate professional information so suitable users can book me.
- As a therapist, I want my availability to be represented correctly so I avoid scheduling conflicts.
- As a therapist, I want session and booking statuses to be reliable so I know which users are expected and paid.

## Operations Stories

- As an operations user, I want therapist profiles to be manageable outside the mobile app so I can maintain quality control.
- As an operations user, I want to inspect booking and payment states so I can resolve support issues.
- As an operations user, I want failed or ambiguous payment cases to be traceable so bookings can be corrected without guesswork.

## Acceptance Notes

A story should not be considered done unless:

- the happy path works end to end
- the user sees a clear state at the end of the flow
- obvious failure states are handled
- the implementation matches `requirements.md`
- the implementation stays within `scope.md`
